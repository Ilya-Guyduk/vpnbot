# Маршрутизация в v2rayNG — российские сайты напрямую

По умолчанию v2rayNG пускает **весь трафик** через VPN. Это работает, но имеет минусы: российские сайты (Сбер, Госуслуги, Яндекс, RuTube) тормозят или вообще требуют отключить VPN, потому что блокируют иностранные IP. Также все ваши локальные сервисы (домашний роутер, принтер, NAS) перестают видеться.

Решение — настроить **маршрутизацию**: российский трафик идёт напрямую, всё остальное — через VPN. В этом гайде разберём два способа: простой пресет и продвинутые правила.

### Что нам понадобится

Используем **российские geo-файлы** от проекта [runetfreedom](https://github.com/runetfreedom/russia-v2ray-rules-dat). Это списки доменов и IP-адресов, которые автоматически обновляются каждые 6 часов на основе данных РКН.

Что есть в этих файлах:

- **geosite-ru-only.dat** — домены: всё российское (.ru, .рф, банки, госуслуги), плюс категории Discord, YouTube, OpenAI и т.д.
- **geoip-ru-only.dat** — IP-диапазоны: российские сети, заблокированные сети, приватные сети (192.168.x.x)

> **Важно:** для мобильных используем именно `ru-only`-версии файлов. Полные `geoip.dat`/`geosite.dat` слишком большие (десятки мегабайт), на телефоне не нужны.

### Способ 1 — простой пресет

Подходит большинству. Подключаем готовый набор правил без ручных настроек.

#### Шаг 1. Загружаем geo-файлы в v2rayNG

1. Откройте v2rayNG
2. Зайдите в боковое меню (три полоски слева сверху) → «Настройки»
3. Найдите пункт **«Asset files»** (файлы ресурсов)
4. Нажмите «+» → «Add URL»

**Добавьте первую ссылку:**

- URL: `https://github.com/runetfreedom/russia-v2ray-rules-dat/releases/latest/download/geosite-ru-only.dat`
- Remarks: `geosite-ru-only`

**Затем добавьте вторую:**

- URL: `https://github.com/runetfreedom/russia-v2ray-rules-dat/releases/latest/download/geoip-ru-only.dat`
- Remarks: `geoip-ru-only`

> **Внимание!** Поле `Remarks` — это не просто комментарий. v2rayNG использует его как имя файла. Если впишете `Remarks` неправильно — правила не сработают.

После добавления нажмите на иконку ☁️ облака сверху — файлы скачаются.

#### Шаг 2. Импортируем правила маршрутизации

Скопируйте этот JSON в буфер обмена:

```
[{"enabled":true,"locked":false,"outboundTag":"direct","protocol":["bittorrent"],"remarks":"Торренты напрямую"},{"enabled":true,"ip":["ext:geoip-ru-only:private"],"locked":false,"outboundTag":"direct","remarks":"Локальная сеть напрямую"},{"domain":["ext:geosite-ru-only:private"],"enabled":true,"locked":false,"outboundTag":"direct","remarks":"Приватные домены напрямую"},{"enabled":true,"ip":["ext:geoip-ru-only:ru-blocked"],"locked":false,"outboundTag":"proxy","remarks":"Заблокированные сети через VPN"},{"domain":["ext:geosite-ru-only:ru-blocked"],"enabled":true,"locked":false,"outboundTag":"proxy","remarks":"Заблокированные домены через VPN"},{"enabled":true,"ip":["geoip:ru"],"locked":false,"outboundTag":"direct","remarks":"Российские IP напрямую"},{"domain":["geosite:category-ru"],"enabled":true,"locked":false,"outboundTag":"direct","remarks":"Российские домены напрямую"},{"enabled":true,"locked":false,"outboundTag":"proxy","port":"0-65535","remarks":"Всё остальное через VPN"}]
```

Теперь в v2rayNG:

1. Боковое меню → «Routing settings»
2. Нажмите три точки справа сверху → **«Import ruleset from clipboard»**
3. Подтвердите импорт

#### Шаг 3. Активируйте режим маршрутизации

1. В главном экране v2rayNG найдите выпадающий список с режимом (обычно «Global»)
2. Выберите «Custom rules» (или «Custom»)
3. Подключитесь как обычно

Готово! Теперь Сбер, Госуслуги и Яндекс работают напрямую, а заблокированные сайты — через VPN.

### Способ 2 — продвинутые правила

Подходит для тех, кто хочет точно контролировать что куда идёт. Например: Telegram через VPN, торренты вообще заблокировать, Spotify через прокси несмотря на geoip.

#### Принципы написания правил

Правила в v2rayNG проверяются **сверху вниз**. Первое подходящее правило применяется, остальные игнорируются. Поэтому порядок важен.

Каждое правило имеет три ключевых поля:

- **domain / ip / port / protocol** — что попадает под правило
- **outboundTag** — куда отправлять: `direct` (напрямую), `proxy` (через VPN), `block` (заблокировать)
- **remarks** — название правила (для удобства)

#### Полезные категории доменов

Из стандартной базы v2rayNG (всегда доступны):

- `geosite:category-ru` — российские домены
- `geosite:google` — все домены Google
- `geosite:youtube` — домены YouTube
- `geosite:telegram` — домены Telegram
- `geosite:netflix` — Netflix
- `geosite:category-ads-all` — реклама и трекеры
- `geoip:ru` — российские IP
- `geoip:private` — локальные сети

Из подгруженных russia-v2ray-rules:

- `ext:geosite-ru-only:ru-blocked` — заблокированные домены РКН
- `ext:geoip-ru-only:ru-blocked` — заблокированные IP-сети

#### Примеры правил

**Заблокировать рекламу:**

```
{"domain":["geosite:category-ads-all"],"enabled":true,"outboundTag":"block","remarks":"Блокировка рекламы"}
```

**Telegram строго через VPN (на случай если в России начнут резать):**

```
{"domain":["geosite:telegram"],"enabled":true,"outboundTag":"proxy","remarks":"Telegram через VPN"}
{"ip":["geoip:telegram"],"enabled":true,"outboundTag":"proxy","remarks":"Telegram IP через VPN"}
```

**Конкретный домен в обход VPN (пример: avito.ru):**

```
{"domain":["domain:avito.ru"],"enabled":true,"outboundTag":"direct","remarks":"Avito напрямую"}
```

#### Чем отличается domain: от full:

- `domain:example.com` — этот домен и все поддомены (mail.example.com, www.example.com и т.д.)
- `full:example.com` — только точное совпадение
- `regexp:^.+\.ru$` — регулярное выражение (для гиков)

### Как это работает

Когда вы открываете сайт, v2rayNG для каждого запроса делает следующее:

1. Получает домен или IP назначения
2. Проверяет правила одно за другим, сверху вниз
3. Находит первое подходящее — применяет его outbound (direct / proxy / block)
4. Если ни одно не подошло — применяется правило по умолчанию (обычно «всё через proxy»)

Поэтому **правила-исключения должны быть выше** общих. Если поставить «всё через VPN» первым правилом — все остальные правила окажутся бесполезными.

### Решение проблем

#### Все сайты не работают после настройки

Скорее всего одно из:

- Asset files не загрузились — зайдите в Settings → Asset files и нажмите облако ☁️
- В поле Remarks указано неправильное имя — должно быть точно `geosite-ru-only` и `geoip-ru-only` без расширения
- Не выбран режим «Custom rules» в главном экране

#### Часть российских сайтов всё равно идёт через VPN

Geo-файлы покрывают большинство российских сайтов, но не все. Добавьте отдельное правило для конкретного сайта:

```
{"domain":["domain:название-сайта.ru"],"enabled":true,"outboundTag":"direct","remarks":"Название сайта напрямую"}
```

И поставьте его выше правила «Российские домены напрямую».

#### YouTube/Discord не работают

Эти сервисы периодически блокируются по-разному. Если у вас YouTube не открывается даже через VPN, попробуйте принудительно отправлять его через прокси. Добавьте правило `geosite:youtube` → `proxy` в самом верху списка.

#### Хочу всё откатить обратно

В главном экране v2rayNG поменяйте режим обратно на «Global». Все ваши правила останутся, но не будут применяться. Чтобы удалить — Routing settings → выберите правила → удалить.

---

*Если правила работают неожиданно — включите логи (Settings → Enable logs) и посмотрите, какие правила срабатывают для каждого запроса.*
